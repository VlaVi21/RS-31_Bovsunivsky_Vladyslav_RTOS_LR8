**Завдання до лабораторної роботи**: 

Варіант 2. Реалізувати чотири потоки, які керують світлодіодами PD12–PD15 як «світлофор». 
Кнопки PA0–PA3 змінюють тривалість сигналів зеленого, жовтого, червоного та синього 
світлодіодів.

---
### 👨🏼‍💻Функціонал:

**1. Multi-threading (FreeRTOS Tasks):** Реалізація 4-х незалежних потоків для керування світлодіодами PD12–PD15. Кожен потік відповідає за свою "фазу" світлофора.

**2. Inter-Task Synchronization (Semaphores):** Використання бінарних семафорів для передачі "естафети" між потоками. Це забезпечує сувору послідовність роботи світлодіодів (червоний -> жовтий -> зелений -> синій).

**3. Message Queuing (IPC):** Передача нових значень затримки (тривалості фази) від обробника переривань кнопки (ISR) до відповідного потоку через черги xQueue.

**4. Interrupt-Driven Control:** Зміна параметрів системи в реальному часі через кнопки PA0–PA3 без блокування роботи планувальника задач.

---
### 🔵Підключення:

**1. Кнопки (Входи керування):**

- PA0, PA1, PA2, PA3 — налаштовані як GPIO_EXTI (External Interrupt).

- Режим: Rising Edge (спрацювання при натисканні).

- Внутрішня підтяжка: Pull-up.

**2. Світлодіоди (Виходи):**

- PD12 (Red), PD13 (Orange), PD14 (Green), PD15 (Blue).

- Режим: GPIO_Output.

**3. Приблизна схемотехнічна реалізація:**

 -1.1 GND - [-LED+] - [резистор 325R] - PB15

- 1.2 GND - [-LED+] - [резистор 325R] - PB14
  
- 1.3 GND - [-LED+] - [резистор 325R] - PB13
  
- 1.4 GND - [-LED+] - [резистор 325R] - PB12

- 2.1 GND - [-Key+] - PA0 (Pull_up)

- 2.2 GND - [-Key+] - PA1 (Pull_up)

- 2.3 GND - [-Key+] - PA2 (Pull_up)

- 2.4 GND - [-Key+] - PA3 (Pull_up)

### 🛠️ Налаштування периферії

У цій роботі реалізовано складну взаємодію між апаратними перериваннями та ядром ОС.

**Переваги обраної архітектури:**

**- Ефективність:** Завдяки семафорам потоки, що чекають своєї черги, перебувають у стані Blocked і не споживають ресурси процесора.

**- Безпека даних:** Використання черг гарантує, що дані від кнопок будуть передані коректно, навіть якщо потік у цей момент зайнятий іншою операцією.

**- Масштабованість:** Структура дозволяє легко додавати нові фази або змінювати логіку роботи кожного світлодіода окремо.

### ⁉️Як запустити:
1. Відкрийте проект у STM32CubeIDE/Keil.
2. Скомпілюйте проект (Build) та прошийте плату через ST-Link.
3. Після ініціалізації перший семафор xSemNext[0] примусово віддається (Give), щоб запустити цикл.
4. Потік RED бере семафор -> виконує роботу -> віддає семафор потоку YELLOW.
5. При натисканні кнопки на PA0, функція HAL_GPIO_EXTI_Callback відправляє нове значення затримки в xQueueLED[0].
6. Коли потік RED наступного разу отримає доступ до процесора, він забере нове значення з черги та змінить час свого горіння.
